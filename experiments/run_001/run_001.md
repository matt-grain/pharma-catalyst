## Baseline Model (Before Modification)

The starting point that agents were given to improve:

### Code

```python
def smiles_to_fingerprint(smiles: str, radius: int = 2, n_bits: int = 1024) -> np.ndarray:
    """
    Convert a molecule (as SMILES string) to a numerical feature vector.

    SMILES = "Simplified Molecular Input Line Entry System"
    Example: "CCO" = ethanol (C-C-O), "c1ccccc1" = benzene (aromatic ring)

    Morgan Fingerprints:
    - Look at each atom and its neighborhood (up to 'radius' bonds away)
    - Hash these neighborhoods into a fixed-size bit vector
    - If two molecules share substructures, they'll have similar fingerprints

    Think of it like: "What building blocks does this molecule contain?"

    Args:
        smiles: Molecule as SMILES string (e.g., "CCO" for ethanol)
        radius: How far to look around each atom (2 = up to 2 bonds away)
        n_bits: Size of output vector (1024 = 1024-dimensional fingerprint)

    Returns:
        numpy array of 0s and 1s (1024 bits) representing molecular structure
    """
    # Parse SMILES string into molecule object
    mol = Chem.MolFromSmiles(smiles)
    if mol is None:
        return np.zeros(n_bits)  # Invalid molecule -> all zeros

    # Generate Morgan fingerprint
    generator = rdFingerprintGenerator.GetMorganGenerator(radius=radius, fpSize=n_bits)
    fp = generator.GetFingerprintAsNumPy(mol)
    return fp


# Model: RandomForest for regression
model = RandomForestRegressor(
    n_estimators=100,   # Build 100 decision trees, average their predictions
    max_depth=10,       # Each tree can have at most 10 levels (prevents overfitting)
    random_state=42,    # Fixed seed for reproducibility
    n_jobs=-1           # Use all CPU cores for parallel training
)
# Why RandomForest?
# - Works well with sparse, high-dimensional data (1024-bit fingerprints)
# - No feature scaling needed (trees don't care about magnitude)
# - Robust to overfitting with default parameters
# - Fast training on small datasets
```

## What the Agents Did

### 1. Hypothesis Agent (Research Scientist)

**Proposal:** Augment the Morgan fingerprints with four physicochemical descriptors:
- `MolLogP` - Lipophilicity (octanol-water partition coefficient)
- `MolWt` - Molecular weight
- `NumRotatableBonds` - Flexibility indicator
- `HeavyAtomCount` - Size/complexity indicator

**Reasoning (verbatim from agent):**
> Aqueous solubility (logS) is primarily determined by a molecule's lipophilicity (LogP) and size (Molecular Weight), as established in the original ESOL (Estimated SOLubility) paper by Delaney. While Morgan fingerprints are excellent at capturing local structural fragments, they do not explicitly encode these global physicochemical properties. By concatenating these descriptors with the fingerprints, the model can directly leverage the physical drivers of solubility, which is especially beneficial for a small dataset like ESOL where learning these relationships from sparse bit vectors alone is difficult.

**Why this is smart:**
- The agent correctly identified that Morgan fingerprints capture *local* structure but miss *global* properties
- Referenced the original ESOL paper (Delaney 2004) - showing domain knowledge
- Recognized the small dataset limitation and proposed a solution

### 2. Model Agent (ML Engineer)

**Implementation:** Modified `train.py` to:

```python
# Added import
from rdkit.Chem import Descriptors

# Modified featurization function
def smiles_to_fingerprint(smiles: str, radius: int = 2, n_bits: int = 1024) -> np.ndarray:
    """Convert SMILES to Morgan fingerprint plus physical descriptors."""
    mol = Chem.MolFromSmiles(smiles)
    if mol is None:
        return np.zeros(n_bits + 4)  # +4 for descriptors

    # Morgan fingerprint
    generator = rdFingerprintGenerator.GetMorganGenerator(radius=radius, fpSize=n_bits)
    fp = generator.GetFingerprintAsNumPy(mol)

    # Physicochemical descriptors
    descriptors = np.array([
        Descriptors.MolLogP(mol),
        Descriptors.MolWt(mol),
        Descriptors.NumRotatableBonds(mol),
        Descriptors.HeavyAtomCount(mol)
    ])

    return np.concatenate([fp, descriptors])
```

**Key changes:**
- Feature vector expanded from 1024 bits to 1028 (1024 + 4 descriptors)
- Descriptors normalized implicitly by RandomForest (tree-based models handle different scales)
- Graceful handling of invalid molecules (zeros)
